Learning Notes---Selective Search

Posted on 2015-10-12   |   In object recognition   |  

Selective Search is used to find possible location bounding boxes for object detection. It combines the strength of exhaustive search and segmentation, and use different strategies based on low level features to sample the origin bounding boxes. Compared with other methods, it produce less bounding boxes and improve the accuracy.

Since the results are relatively good, and are class-independent, recent papers focusing on object localization and segmentation with CNN use it to preprocess images.

Source: Selective Search for Object Recognition

Generate origin regions: Selective Search by Hierarchical Grouping

Use method in Efficient Graph-Based Image Segmentation to get initial regions, then merge similar regions, and do it iteratively to obtain candidate regions in different scales until the whole image becomes a single region.

Different strategies to mearsure similarity

  1. Color similarity

    For each region, generate one-dimensional color histograms for each colour channel using 25 bins. And the histograms is normalized by L1 norm. Similarity is measured using the histogram intersection.

    In order to take all invariants into consideration, use 8 different color spaces.

  2. Texture similarity

    Use fast SIFT-like(not SIFT) to represent it. Specificly, take Gaussian derivatives in eight orientations using Ïƒ = 1 for each colour channel, and extract a histogram using a bin size of 10 for each channel and each direction. Similarity is alse measured by the histogram intersection.

  3. Size similarity

    To prevent small regions being eaten by large regions, it encourages small regions to merge first. Similarity is measured by:

    $${S_{size}}({r_i},{r_j}) = 1 - {{size({r_i}) + size({r_j})} \over {size(im)}}$$

  4. Distance similarity

    This shows how how well region ri and rj fit into each other. Each bounding region is surrounded by a tight bounding box, and the similarity is measured by the area not covered by the regions in the bounding boxes.

  5. Complementary Starting Regions

    Different starting region is generated by different color space and the threshold. Each has different invariance properties. Use different combination of it to improve the results.
    To be specific, use differcent color space indvidualy, and simply take all the bounding boxes.

Combining Locations

Order the combined object hypotheses set based on the order in which the hypotheses were generated in each individual grouping strategy. That means order the hypotheses set according to the scale. To prevent too heavy emphasis large regions, multiply the large region positioin values with a random number in range [0,1].

Experimental results

Selective search is highly effective for generating object locations in terms of regions. But there is a trade-off between quality and quantity of object hypothesis and the object recognition accuracy. The article indicates that at a reasonable 10,000 locations, their object hypothesis set is close to optimal for the Bag-of-Words recognition system. Maybe it can be a refrence when applying it to CNN.
